1. What is the difference between map, filter, forEach and find?

Ans:JavaScript's higher-order functions map, filter, forEach, and find are all used to change arrays. Here's a quick explanation of how they're different:

map: This method makes a new array by calling a given function on each element of the original array. The current element, its index, and the original array are passed to the function. The function returns a value, which is added to the new array. The new array will always have the same number of elements as the original array.


filter: This method makes a new array by removing from the original array any elements that don't pass a test function. The current element, its index, and the original array are passed to the function. It should return a boolean value that says whether or not the element should be added to the new array. The size of the new array might be less than the size of the old array.


forEach: This method goes through each array element and runs a function for each one. The current element, its index, and the original array are passed to the function. The function doesn't return anything, though. It is mostly used for side effects, like writing to a log or changing a variable.


find: This method returns the first element in the array that passes a given test function. The current element, its index, and the original array are passed to the function. It should return a boolean value that says whether or not the element passes the test. Undefined is returned if no element passes the test.

2. What is the difference local storage, session storage, and cookies?

Ans: Web browsers offer localStorage, sessionStorage, and cookies as ways to store data on the client side. Here's a quick explanation of how they're different:

localStorage: This mechanism lets web apps store key-value pairs locally on a user's browser. Even if the user closes the browser window or restarts the computer, the data stored in localStorage will still be there. The data is available in multiple tabs and windows, and the same JavaScript code can be used to get to it.


sessionStorage: This is similar to localStorage, but the data stored in sessionStorage is tied to a specific session. Data stored in sessionStorage is only available in the current window or tab and is deleted when the window or tab is closed. But if the user opens a new tab or window, the data stored in sessionStorage will be available in the new tab or window.


Cookies are small text files that a web server puts on a user's browser. They can be used to store small amounts of data, like user preferences or login information, and every request sends them to the server. Cookies can be set to disappear after a certain amount of time or when the user closes the browser.

3. What is event loop and how does it works?

Ans: An event loop is a piece of code that is used in event-driven programming, a common way to build websites today. It is used to manage and coordinate the different activities and events that happen in an application. At a high level, an event loop works by constantly watching a queue of events or tasks, such as user inputs or network requests. When an event happens, it is added to the queue, and the event loop processes the queue in a sequential, non-blocking way. The event loop starts by looking at the queue to see if any events or tasks are waiting to be done. If none of them happen, it waits until one does. Once an event is found, the event loop will handle it by calling the right callback function or event handler. The handler function may then start more tasks or events, which will also be added to the queue to be processed. One important thing about an event loop is that it doesn't stop the main execution thread. This means that while the event loop is waiting for events or tasks to finish, other parts of the application can still run. This makes it possible for applications to work quickly and well. Overall, the event loop is an important part of event-driven programming because it makes it possible for applications to work quickly and effectively.
